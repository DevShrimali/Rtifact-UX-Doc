Deep dive — the strategic gap

Tools like Dash0 & SigNoz are highly technical, catering only to engineers but failing to communicate business value (ROI / FinOps) to leadership.

Tools like PointFive cater well to executives with cost metrics, but lack the granular, actionable telemetry SREs need to actually fix issues.

Rtifact's positioning: a minimalist, typography-led UI presents high-level ROI to Raj (VP) on the overview, while Priya (SRE) drills down via breadcrumbs into unified telemetry — both personas served by one platform.

Discover / 05

User journey map

The incident resolution workflow, per persona: what they do, feel, and where stress peaks — and the exact UX gain designed for each stage.

Deep dive — transforming pains into gains

1 · Triage stage. Pain: context switching across tabs. Gain: the Unified Telemetry view locks logs, metrics and traces into the same temporal context.

2 · Investigation stage. Pain: playing detective manually. Gain: the Incident Causal Chain & SLO Budget view provides a direct cause → effect path, a confidence-ranked "What changed" list, and error-budget gauges (with a fallback warning if Deployment Intelligence is disconnected).

3 · Reporting stage. Pain: hard to prove value to the VP. Gain: automated audit logs feed directly into the Executive ROI dashboard.

Define / 06

Core problem — human-centered design scope

"How might we empower engineers to instantly resolve cloud issues without context-switching, while visually proving ROI value to leadership through a radically simplified interface?"
Current state

Engineers spend 70% of incident time correlating logs and metrics manually. The core issue isn't a lack of data — it's a lack of synthesis.

Rtifact state

AI auto-correlates telemetry, shifting engineer focus from "detective work" to "approving the fix."

Design principle

Delete until we've deleted too much — we'll put it back. Every screen answers one question per persona above the fold; everything else is progressively disclosed.

Pattern spec — Ask AI contextual assistant

One reusable pattern, never re-invented per page. Always consistent: a bottom chat input bar, inline context history that scrolls under (keeping the canvas visible), and a reference-attaching mechanism showing exactly which logs, metrics or clusters are in context.

  • Incident view — remediation mode: proposes shell command patches, script snippets, 1-click execution templates.
  • Metrics view — exploratory mode: explains anomalies, runs correlation SQL/PromQL, analyzes trends.
  • Signals view — suggests query filter syntax, parses complex JSON logs, maps trace spans.
  • Alerts view — helps configure thresholds, group alerts, tune noise reduction.

Sign-off: Matt approved documenting this as one feature with distinct surface behaviors — prevents developer duplication, guarantees one visual standard.

Pattern spec — Incident Causal Chain & SLO Budget view

Cause → Effect → Impact chain: a visual flow mapping the trigger root (e.g. database disk spike) through the immediate side effect (latency spike) to the customer-facing impact.

  • Before / Now metrics — side-by-side comparison of baseline vs. degraded state, rendered in mono.
  • Confidence-scored "What changed" list — deployments, commits and config toggles ranked by correlation score.
  • SLO error-budget bars — budget depletion and burn-rate gauges.

Degraded state: without the Deployment Intelligence agent, the view falls back to time-based event logs; confidence scores are downgraded and the UI urges connecting the agent.

Structure / 08

Reusable module navigation template

One template, three modules. Learn the layout once in Review & Manage and it carries over to Monitor and Support unchanged: KPI row → sub-nav tabs → workspace. Switch modules to see the same skeleton re-skinned by data alone.

app.rtifact.io/#/review
Template contract

Layout constraints

  • KPI row: exactly 3 priority metrics reflecting real-time state — never more.
  • Sub-nav: horizontal borderless tabs, left aligned, maximum 4 sub-views to prevent overflow.
  • Drill-down harmony: level 1 is the sub-nav tab; levels 2–3 render as routed pages with breadcrumbs — never modals.
  • Consistency paradigm: identical skeleton across all three modules, so the learning curve transfers on first use.
Deep dive — per-module navigation spec

Review & Manage · KPI row: Monitored Assets | Compliance Score | Active Findings · Tabs: Asset Inventory, Compliance Audit, Remediation Center, SOC2 Reports.

Monitor · KPI row: Active Incidents | Global MTTR | Noise Reduction Rate · Tabs: Telemetry Signals, Alerts Workspace, Incidents Board, Synthetics Control.

Support · KPI row: Open Cases | Avg SLA Response | Customer CSAT · Tabs: Ticket Queue, System State Correlation, Client Response Panel.

Why: Sunil and Matt want the platform to feel instantly familiar as users switch between modules — the reusable template is that guarantee.
